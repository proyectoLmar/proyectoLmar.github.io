¿Qué es una Base de Datos?

Una base de datos es un conjunto organizado de datos que se almacenan y gestionan de manera eficiente. Se utiliza para almacenar información estructurada y permitir su recuperación y manipulación.

Pasos para Crear una Base de Datos:

  1. Planificación y diseño: Antes de crear una base de datos, define los requisitos y estructura de tus datos.
  2. Selecciona una herramienta: Puedes utilizar sistemas de gestión de bases de datos como MySQL, PostgreSQL o MongoDB.
  3. Creación de tablas: Diseña las tablas y define las relaciones entre ellas.
  4. Operaciones CRUD: Aprende a realizar operaciones de Crear, Leer, Actualizar y Eliminar datos en tu base de datos.

Recursos Adicionales:

  • MySQL: Sistema de gestión de bases de datos relacional.
  • PostgreSQL: Base de datos de código abierto y potente.
  • MongoDB: Base de datos NoSQL orientada a documentos.

Esquema entidad-relación

  1. Identificar la Entidad: Determina cuál es la entidad principal de la tabla. Una entidad representa un objeto del mundo real que tiene importancia para el sistema que estás modelando. Por ejemplo, en una base de datos de estudiantes, una entidad podría ser Estudiante.

  2. Definir los Atributos: Lista los atributos de la entidad. Los atributos son las propiedades o características que describen la entidad. Por ejemplo, para la entidad Estudiante, los atributos podrían ser ID_Estudiante, Nombre, Apellido, Fecha_Nacimiento, y Email.

  3. Establecer la Clave Primaria: Identifica el atributo o conjunto de atributos que sirven como identificador único para cada instancia de la entidad. En el ejemplo de Estudiante, el atributo ID_Estudiante podría ser la clave primaria.

  4. Determinar las Relaciones: Si la tabla está relacionada con otras tablas, identifica las relaciones. Las relaciones pueden ser uno a uno, uno a muchos o muchos a muchos. Por ejemplo, si un estudiante puede estar inscrito en varios cursos, y un curso puede tener varios estudiantes, habría una relación muchos a muchos entre Estudiante y Curso.

  5. Crear el Diagrama ER: Usa un software de diagramación (como Microsoft Visio, Lucidchart, o una herramienta de diagramas ER específica) para dibujar el diagrama. Representa la entidad como un rectángulo. Lista los atributos dentro del rectángulo. Marca la clave primaria subrayando el atributo correspondiente. Dibuja líneas para representar las relaciones con otras entidades, usando notación adecuada.

Creación de tablas

  1. Crear la tabla: Para crear una tabla, utiliza el comando CREATE TABLE, seguido del nombre de la tabla y la definición de sus columnas. Cada columna se define con un nombre, un tipo de datos y, opcionalmente, restricciones (como si es una clave primaria, si puede ser nula, etc.).

  2. Definir las columnas: Las columnas deben tener un tipo de dato adecuado según el tipo de información que almacenarán.(Int para numeros entero, varchar para texto, date para fechas...)

Ejemplo: Supongamos que quieres crear una tabla llamada Estudiantes que almacene el ID del estudiante, su nombre, apellido, fecha de nacimiento y email. El ID del estudiante será la clave primaria.

CREATE TABLE Estudiantes (
ID_Estudiante INT PRIMARY KEY,
Nombre VARCHAR(50) NOT NULL,
Apellido VARCHAR(50) NOT NULL,
Fecha_Nacimiento DATE,
Email VARCHAR(100)
);

Consultas

  1. Agregar condiciones Puedes agregar condiciones a tu consulta utilizando la cláusula WHERE. Esto filtra los resultados basados en una condición específica.

    SELECT campo1, campo2 FROM tabla WHERE condicion;

  2. Ordenar los resultados Puedes ordenar los resultados de tu consulta utilizando la cláusula ORDER BY. Esto te permite especificar un campo y un orden (ascendente o descendente).

    SELECT campo1, campo2 FROM tabla ORDER BY campo ASC

Ejemplo: Supongamos que tienes una tabla llamada Empleados con campos ID, Nombre, Apellido y Salario, y quieres obtener los nombres y apellidos de todos los empleados cuyo salario sea superior a 5000:

SELECT Nombre, Apellido FROM Empleados WHERE Salario > 5000;

Joins

Los joins en SQL se utilizan para combinar filas de dos o más tablas basadas en una columna relacionada entre ellas. Esto es útil para recuperar datos que están distribuidos en múltiples tablas en una base de datos relacional.

  • INNER JOIN: Devuelve filas con coincidencias en ambas tablas.

  • LEFT JOIN: Devuelve todas las filas de la tabla izquierda y coincidencias de la derecha.

  • RIGHT JOIN: Devuelve todas las filas de la tabla derecha y coincidencias de la izquierda.

  • FULL JOIN: Devuelve filas con coincidencias en una de las tablas, completando con NULL donde no hay coincidencias.

  • CROSS JOIN: Devuelve el producto cartesiano de las tablas.

  • SELF JOIN: Une una tabla consigo misma.

  • NATURAL JOIN: Une las tablas basándose en columnas con el mismo nombre.
